## 🧠 What Is This Project?

> **This portfolio website is itself an agentic AI system** — not a static page. The chat interface below is powered by a RAG pipeline built from my actual GitHub repos and knowledge base. It knows my projects better than a static page ever could.

<br/>

<table>
<tr>
<td align="center" width="33%">

### 🤖 AI Digital Twin
A conversational AI that **answers as me** — with my real opinions, real tradeoffs, real uncertainty. Powered by `llama3.2:3b` running 100% locally via Ollama.

</td>
<td align="center" width="33%">

### 🔍 RAG Pipeline
**Hybrid retrieval**: cosine similarity vectors + BM25 keyword matching. Documents are chunked, embedded via `nomic-embed-text`, and stored in a JSON vector store.

</td>
<td align="center" width="33%">

### 👁️ Visitor Intelligence
**Persona detection** from HTTP referrer and UTM params. A recruiter from LinkedIn sees different AI responses than an engineer from GitHub. Zero tracking, pure signal analysis.

</td>
</tr>
<tr>
<td align="center" width="33%">

### 🐙 GitHub Sync
Auto-fetches repos, READMEs, and metadata via **Octokit API**. The AI always knows my latest projects without manual updates.

</td>
<td align="center" width="33%">

### 📡 SSE Streaming
Real-time **token-by-token** chat responses via Server-Sent Events. Markdown rendering with source citations as the AI generates.

</td>
<td align="center" width="33%">

### 💰 Zero Cost
**No API keys. No cloud bills. No vendor lock-in.** Everything runs on `npm run dev` with Ollama. Upgrade path to production when ready.

</td>
</tr>
</table>

## 🔍 How the RAG Pipeline Works

The AI doesn't just generate random text. It **retrieves real context** from my knowledge base before answering.

```
USER QUESTION: "Tell me about your RAG research assistant"
                              │
                              ▼
                    ┌──────────────────┐
                    │  1. EMBED QUERY  │
                    │  nomic-embed-text│
                    │  → 768-dim vector│
                    └────────┬─────────┘
                             │
                ┌────────────┴────────────┐
                ▼                         ▼
      ┌──────────────────┐     ┌──────────────────┐
      │  2. VECTOR SEARCH │     │  3. BM25 KEYWORD │
      │  Cosine similarity│     │  Term frequency   │
      │  (semantic match) │     │  (exact matching) │
      │  Top-10 chunks    │     │  Top-10 chunks    │
      └────────┬──────────┘     └────────┬──────────┘
               │                         │
               └────────────┬────────────┘
                            ▼
                  ┌──────────────────┐
                  │  4. HYBRID MERGE │
                  │  Best of both    │
                  │  Top-5 chunks    │
                  └────────┬─────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │  5. GENERATE     │
                  │  llama3.2:3b     │
                  │  System prompt   │
                  │  + persona       │
                  │  + RAG context   │
                  │  + chat history  │
                  └────────┬─────────┘
                           │
                           ▼
                  ┌──────────────────┐
                  │  6. SSE STREAM   │
                  │  Token by token  │
                  │  + source refs   │
                  │  → Browser       │
                  └──────────────────┘
```

**Why hybrid retrieval?** Vector search finds semantically similar content ("RAG" matches "retrieval augmented generation"). BM25 finds exact keyword matches ("pgvector" matches "pgvector"). Together they catch what either alone would miss.

## 🚀 Quickstart

### Prerequisites

- **Node.js** 18+ ([download](https://nodejs.org))
- **Ollama** ([install](https://ollama.ai)) — local AI runtime
- ~4GB disk space for AI models

### Setup (3 minutes)

```bash
# 1. Clone the repository
git clone https://github.com/aman-bhaskar-codes/portfolio-website.git
cd portfolio-website

# 2. Install dependencies
npm install

# 3. Start Ollama and pull models (in a separate terminal)
ollama serve
ollama pull llama3.2:3b          # Chat model (2GB)
ollama pull nomic-embed-text     # Embedding model (274MB)

# 4. Create your environment file
cp .env.example .env.local

# 5. Ingest the knowledge base into vectors
npx tsx scripts/ingest.ts

# 6. Start the dev server
npm run dev
```

Then open **http://localhost:3000** — the portfolio with AI chat is live! 🎉

### Optional: Enable GitHub Sync

Add your GitHub token to `.env.local` to auto-ingest your repos:

```bash
GITHUB_TOKEN=ghp_your_token_here
```

Then re-run `npx tsx scripts/ingest.ts` to embed your GitHub data.
