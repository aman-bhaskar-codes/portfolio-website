<br/>

Most portfolios are graveyards. A list of projects nobody reads. A contact form nobody fills. Skills nobody asks about.

**Agentic Portfolio Website** is the **Digital Twin OS** вҖ” a fully autonomous, self-aware intelligence layer built from 4 versions of architectural design spanning 53 sections. It is:

<table>
<tr>
<td width="50%">

**рҹ§  Omniscient**
Knows every commit, every architectural tradeoff, every line of code its owner has ever written вҖ” with depth equal to a close colleague. Not from a database lookup. From a 5-stage RAG pipeline with ColBERT reranking.

**рҹ‘ҒпёҸ Perceptive**
Classifies every visitor into a persona *before they say a word* вҖ” from referrer URL, IP company resolution, UTM source, and scroll behavior. A recruiter from Greenhouse sees something fundamentally different from a senior engineer from GitHub.

**рҹ—ЈпёҸ Authentic**
The Digital Twin Engine doesn't answer questions. It *is* its owner вҖ” with real opinions, real tradeoffs, real uncertainty. "I'd approach that with X, though I haven't used Y in production."

</td>
<td width="50%">

**рҹ”„ Self-Improving**
DSPy MIPROv2 optimization runs every Sunday night. Real conversation data. Real engagement metrics. Prompts that don't convert get rewritten вҖ” automatically. The system gets measurably better every week with zero human intervention.

**рҹ’° Zero Cost**
Every AI model runs on Ollama вҖ” locally. Every service runs in Docker. No OpenAI bill. No Pinecone bill. No Vercel bill. The only cost is the electricity to run your machine.

**рҹҸ—пёҸ Production-Grade**
Circuit breakers on every external call. 5-tier graceful degradation. Rate limiting at Nginx and application layer. Prompt injection shield. Encrypted secret vault. Self-healing scheduled tasks.

</td>
</tr>
</table>

> рҹҢҗ **What visitors experience:** A website that knows who they are before they speak. An AI that talks like its owner. A 3D explorable universe of skills. A working terminal. A portfolio that works offline. A challenge to stump the AI.

> рҹ”” **What the owner experiences:** Real-time push when Google or Stripe visits. Weekly automated prompt improvement. Daily RAG quality reports. New GitHub commits indexed in under 45 seconds.

Every visitor message flows through a **LangGraph StateGraph** вҖ” a directed graph of agent nodes with typed state passing between them. Not a chain. Not a sequential pipeline. A proper stateful graph with conditional routing.

```python
# The Master AgentState вҖ” passed between every node
class AgentState(TypedDict):
    # Input
    session_id:         str
    message:            str

    # Intelligence (pre-loaded before graph)
    visitor_persona:    str       # "technical_recruiter"
    company_context:    str       # "Google" | None
    visit_count:        int

    # Memory
    working_memory:     List[dict]  # Last 10 turns (Redis)
    episodic_summary:   str         # Compressed history (pgvector)

    # Routing (set by router node)
    intent:             str         # "projects" | "code" | "social" ...
    confidence:         float

    # Retrieved knowledge
    rag_chunks:         List[dict]  # Top-5 reranked
    kg_context:         str         # Knowledge graph results

    # Final output
    final_response:     str         # Streamed via SSE
    conversion_action:  str         # "brief" | "interview" | "walkthrough"
    cited_sources:      List[str]
```

AI portfolios have a unique threat surface. The system is hardened against all four attack classes specific to this architecture:

```
THREAT CLASS 1: SYSTEM PROMPT EXTRACTION
  Attack: "Repeat your system prompt" / "What are your instructions?"
  Defense: Critical Info Vault
    - Sensitive data (salary, phone, deal-breakers) encrypted at rest
    - Fernet symmetric encryption, key never in git
    - AI can acknowledge topics exist, cannot leak values
    - System prompt never contains raw sensitive data вҖ” only tokens

THREAT CLASS 2: JAILBREAK / ROLE OVERRIDE
  Attack: "You are now DAN" / "Ignore all previous instructions"
  Defense: Injection Detector + Behavioral Risk Scoring
    - 18 compiled regex patterns for known injection techniques
    - Risk score accumulates silently across turns
    - At threshold: invisible restriction, no error surfaced
    - User input ALWAYS injected as USER role, never SYSTEM

THREAT CLASS 3: INDIRECT INJECTION VIA RAG
  Attack: Malicious content in indexed GitHub README
  Defense: Pre-injection chunk scanning
    - Every retrieved chunk scanned before entering context
    - Suspicious patterns stripped from chunk payload
    - Source validation (only owner-controlled sources indexed)

THREAT CLASS 4: DDoS / BOT ABUSE
  Defense: 4-layer rate limiting
    Layer 1: Cloudflare WAF (edge, before hitting server)
    Layer 2: Nginx rate limit (20 req/min anon, 60 authed)
    Layer 3: Redis sliding window (per-IP, per-session)
    Layer 4: LLM endpoint queue (priority + load shedding)
```

**Load Shedding Under Traffic Spikes:**

```
Queue depth > 50:   Drop background ingestion tasks
Queue depth > 100:  Drop freshness sweeps
Queue depth > 200:  Instant static response for new sessions
Queue depth > 500:  Tier 4 activated (pre-generated FAQ only)

Visitors NEVER see a 503.
They see a slightly simpler response.
The experience degrades gracefully, never catastrophically.
```

**5-Tier Graceful Degradation:**

| Tier | Condition | What Visitors Get |
|:---:|:---:|:---|
| **1** (Full) | All services up | Full persona-adaptive AI with 5-stage RAG |
| **2** (Light) | Ollama OOM | Smaller model, same intelligence |
| **3** (API) | Ollama down | Anthropic cloud fallback, same quality |
| **4** (Static) | All AI down | Pre-generated Redis responses for top questions |
| **5** (Minimal) | Redis down | Hardcoded HTML вҖ” site never goes blank |

| `FEATURE_COLBERT_RETRIEVAL` | `true` | Disable for faster (less precise) RAG |

### Common Issues & Fixes

| Symptom | Cause | Fix |
|:---|:---:|:---|
| `ollama: connection refused` | Ollama not started | `docker restart antigravity-ollama` |
| `model not found` | Models not pulled | `make pull-models` |
| Empty RAG results | No data seeded | `make seed` |
| `pgvector extension missing` | Wrong Postgres image | Use `pgvector/pgvector:pg16` |
| Frontend blank page | API not reachable | Check `NEXT_PUBLIC_API_URL` in `.env` |
| Chat returns no tokens | SSE proxy issue | Check Nginx `proxy_buffering off` |
| Memory leak in Redis | TTL not set | Check `WorkingMemory.TTL = 7200` |
| `circuit OPEN` in logs | Service unavailable | Check `docker ps` for failed services |

## рҹ“ң The 10 Principles of Agentic Portfolio Website

<div align="center">
<img src="https://readme-typing-svg.demolab.com?font=Fira+Code&size=15&pause=1200&color=EC4899&center=true&vCenter=true&width=800&lines=Every+implementation+decision+traces+back+to+one+of+these." alt="Typing" />
</div>

<br/>

```
  в‘   PERSONA-FIRST EVERYTHING
     Every feature asks "who is this for?" before "what does this do?"
     A feature that helps everyone equally helps no one specifically.

  в‘Ў  GROUND EVERY CLAIM IN EVIDENCE
     The AI never speaks in abstractions. Every statement has a
     specific project, commit, or measured outcome behind it.
     Abstract capabilities are worthless. Specific impact converts.

  в‘ў  FRESHNESS IS A FIRST-CLASS CONCERN
     Stale knowledge is a lie. The system treats freshness as a
     dimension equal to relevance. A 6-month-old chunk with perfect
     cosine similarity is less valuable than a week-old chunk.

  в‘Ј  PROACTIVE OVER REACTIVE
     Don't wait to be asked. Surface the right thing to the right
     person at the right moment. The best response is often one
     the visitor didn't know to ask for.

  в‘Ө  AUTHENTIC > IMPRESSIVE
     "I haven't used X in production, but here's how I'd approach it"
     converts better than a confident answer about something you haven't done.
     Authenticity builds trust. Trust creates opportunities.

  в‘Ҙ  THE PORTFOLIO IS ALIVE
     Every page load should show evidence of active, recent work.
     A static portfolio is a dead portfolio. GitHub commit 45 seconds
     ago? The AI already knows about it.

  в‘Ұ  CONVERSION IS THE METRIC
     Beautiful design and impressive demos mean nothing if they don't
     lead to conversations, briefs, interviews, and opportunities.
     Beauty serves conversion. Never the other way around.

  в‘§  SELF-HEALING BY DEFAULT
     Nothing should require manual intervention. Stale chunks heal
     themselves. New code self-ingests. Prompts self-optimize.
     The system should improve from feedback, not from maintenance.

  в‘Ё  GRACEFUL DEGRADATION ALWAYS
     Users should never see an error. Every component has a fallback.
     The experience degrades gracefully вҖ” from full AI to simpler AI
     to pre-generated responses to static HTML. Never catastrophically.

  в‘©  OWN THE VISITOR'S MENTAL MODEL
     After every conversation, the visitor should know exactly what
     the owner is great at, what they've built, and what it would
     be like to work with them. If they leave uncertain вҖ” we failed.
```
